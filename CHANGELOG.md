# Changelog

All notable changes to Quorum are recorded here. This project follows semantic versioning.

## [0.2.0]

Refinements from the first dry-run.

- Faithful Representation Gate: added guidance for a labeled "function lens" when no single well-documented person fits a needed capability.
- Interview: added a stop condition, and a three-question fit check.
- Prioritization framework: defined expert-consensus as a confidence-weighted count with an explicit method.
- Portable prompt: produce sections inline when there is no filesystem, and run in one sitting on a plain chat LLM.
- Trust and compliance panel now force-triggers on health, money, minors, or regulated or affiliate claims.
- Board sizing: minimum about twelve seats across at least six seat types, comfortable up to twenty even for a solo operator, quality and diversity over size.

## [0.1.0]

First working version.

- The Quorum loop: fit check, interview, seat the board, audit, deliberation, synthesis, prioritized backlog and roadmap, prioritization framework, PRD handoff, packaging.
- The Faithful Representation Gate, with representation-confidence labels.
- The council-weighted prioritization framework, persisted and overrulable.
- The operating contract for unattended runs.
- Portable prompt for any LLM, plus a Claude Code command and plugin manifest.
- Three illustrative examples.
